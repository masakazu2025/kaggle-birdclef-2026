# Mel Spectrogram

## Spectrogramとは

STFTの結果を2次元画像として可視化したもの。横軸=時間、縦軸=周波数、色の濃さ=音の強さ。

---

## MelスケールとMel Spectrogram

### 問題点
通常のスペクトログラムは周波数が線形スケール（100Hz, 200Hz, 300Hz...）。しかし人間の聴覚は**対数的**で、低音域の違いに敏感で高音域の違いには鈍感。

### Melスケール
人間の聴覚特性に合わせた非線形な周波数スケール。低周波数は細かく、高周波数は粗く分割する。

```
線形スケール: 100, 200, 300, 400, 500 ... Hz（等間隔）
Melスケール:  100, 160, 240, 350, 490 ... Hz（低域が細かい）
```

**Mel Spectrogram** = STFTの結果をMelスケールのフィルタバンクで圧縮したもの。

---

## dB変換（power_to_db）

音のエネルギーは非常に広いダイナミックレンジを持つ（弱い音と強い音で10万倍以上違う）。そのまま使うとモデルが学習しにくいため、対数変換（dB）する。

```python
log_spec = 10.0 * torch.log10(S.clamp(min=1e-10))
```

`top_db=80`：最大値から80dB以上小さい部分を切り捨て。極端に弱い音（背景ノイズ）を除去する。

---

## 正規化

```python
mins = mel_spec.min()
maxs = mel_spec.max()
mel_spec = (mel_spec - mins) / (maxs - mins + 1e-7)  # 0〜1に正規化
```

サンプルごとに0〜1に正規化することで、音量の違うサンプル間でモデルが一貫した特徴を学習できる。

## パラメータまとめ

| パラメータ | 現在値 | 意味 |
|---|---|---|
| `sr` | 32000 | サンプリングレート。1秒あたりのサンプル数 |
| `n_fft` | 2048 | FFT窓サイズ |
| `hop_length` | 512 | 窓のシフト幅 |
| `n_mels` | 256 | Melフィルタの数（周波数解像度） |
| `f_min` | 20 | 分析する最低周波数（Hz） |
| `f_max` | 16000 | 分析する最高周波数（Hz） |
| `top_db` | 80 | ダイナミックレンジの上限（dB） |
| `target_size` | (256, 256) | モデル入力サイズにリサイズ |

---

## BirdCLEFでの改善余地

- **`f_min`を上げる**: 鳥の鳴き声は一般的に500Hz以上。20Hzのまま低音域のノイズも含めている
- **`n_mels`を増やす**: より細かい周波数情報を拾えるが、メモリとGPU時間が増える
- **`f_max`を下げる**: 種によっては8000Hz以下が主要な成分のケースもある

これらの値はDiscussionで上位勢の設定を参考にするのが効率的。
